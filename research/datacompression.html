Data Compression for Inventory Data Optimization

My current research focuses on optimizing storage for inventory data, targeting both text-based records and product images to reduce costs in cloud-based systems like AWS S3 and Aurora. I’m exploring a combination of lossless compression techniques, such as gzip for textual data, and lossy methods, like JPEG for images, to achieve high compression ratios while maintaining data integrity and usability.

The project involves developing and testing compression workflows using Python, integrating algorithms like Huffman coding and Lempel-Ziv-Welch (LZW) for lossless compression, and evaluating JPEG and WebP for lossy image compression. I’m prioritizing a balance between compression ratio and processing speed to ensure scalability for growing datasets, with experiments conducted on AWS EC2 instances to simulate real-world cloud environments.

A key aspect of this research is ensuring security and compliance with standards like GDPR and CCPA. I’m incorporating encryption (e.g., AES) into the compression pipeline to protect sensitive inventory data and verifying that compressed data meets regulatory requirements. Initial results show that gzip achieves up to 70% compression for text records, while JPEG reduces image sizes by 80% with acceptable quality loss for inventory purposes.

Future steps include benchmarking compression performance across larger datasets, optimizing decompression speeds for real-time access, and exploring machine learning-based compression models to further enhance efficiency. This work aims to provide cost-effective, secure, and scalable storage solutions for inventory management systems.
